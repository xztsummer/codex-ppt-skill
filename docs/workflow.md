# 标准工作流

## 总览

Codex PPT 的工作流强调阶段确认。它不会一上来直接生成整套 PPT，而是先确认大纲、风格、图片后端和样张，降低返工成本。

## 阶段 1：阅读输入材料

agent 会先理解：

- 主题和核心观点
- 目标受众
- 演示目标
- 页数要求
- 必须包含或排除的内容
- 是否有必需图片素材

## 阶段 2：确认大纲

agent 会生成 `outline.md`，通常包括：

- Slide 编号
- 每页标题
- 3-5 个要点
- 页面角色，如封面、目录、概念解释、流程、对比、数据证据、总结等
- 可选视觉想法
- 必需图片素材及其用途

大纲确认前，不应生成正式 slide 图片、`speech.md` 或 `.pptx`。

## 阶段 3：确认视觉风格

agent 会给出 2-3 个风格方向，例如：

- 清爽专业风
- 科研答辩风
- 创意杂志风
- 电子墨水杂志风
- 数据仪表盘风
- 手绘技术解释风

选择风格后，整套 PPT 应保持统一视觉语言，但每页版式可以根据内容变化。

## 阶段 4：确认图片生成后端

优先使用 Codex 内置图片生成工具。只有在内置工具不可用、用户明确要求 API/CLI fallback，或当前能力无法满足时，才切换到 `scripts/image_gen.py`。

图片后端确认后，整套 PPT 应保持一致，不应中途切换。

## 阶段 5：生成并确认样张

先生成 1 页样张，检查：

- 文字是否清晰
- 风格是否符合预期
- 页面密度是否合适
- 色彩和排版是否稳定
- 是否适合批量扩展到整套 PPT

样张通过后，再批量生成整套。

## 阶段 6：批量生成与组装

样张确认后，agent 会逐页生成 `origin_image/slide_XX.png`，再生成 `speech.md`，最后用 `assemble_ppt.py` 组装成 `.pptx`。
