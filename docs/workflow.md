# 标准工作流

## 总览

Codex PPT 的工作流强调阶段确认。它不会一上来直接生成整套 PPT，而是先确认大纲、风格、图片后端和样张，降低返工成本。

## 阶段 1：阅读输入材料

agent 会先理解：

- 主题和核心观点
- 目标受众
- 演示目标
- 页数要求
- 必须包含或排除的内容
- 是否有必需图片素材

## 阶段 2：确认大纲

agent 会生成 `outline.md`，通常包括：

- Slide 编号
- 每页标题
- 3-5 个要点
- 页面角色，如封面、目录、概念解释、流程、对比、数据证据、总结等
- 可选视觉想法
- 必需图片素材及其用途

大纲确认前，不应生成正式 slide 图片、`speech.md` 或 `.pptx`。

## 阶段 3：确认视觉风格

agent 会给出 2-3 个风格方向并推荐一个。候选风格来自 12 种内置风格（清爽专业风、科研答辩风、手绘技术解释风、麦肯锡风格、党政红风格、教学课件风等）和你的个人风格库，也可以基于你提供的截图、PDF 或 PPT 复刻风格。完整风格预览参见[风格与个人风格库](styles.md)。

选择风格后，整套 PPT 应保持统一视觉语言，但每页版式可以根据内容变化。

## 阶段 4：确认图片生成后端

优先使用 Codex 内置图片生成工具。只有在内置工具不可用、用户明确要求 API/CLI fallback，或当前能力无法满足时，才切换到 `scripts/image_gen.py`。

图片后端确认后，整套 PPT 应保持一致，不应中途切换。

## 阶段 5：生成并确认样张

先生成 1 页样张，检查：

- 文字是否清晰
- 风格是否符合预期
- 页面密度是否合适
- 色彩和排版是否稳定
- 是否适合批量扩展到整套 PPT

样张通过后，再批量生成整套。

## 阶段 6：批量生成

样张确认后，agent 会逐页生成 `origin_image/slide_XX.png`。在支持子智能体的环境中，会由一个子智能体负责一页并行生成，加快多页产出；所有页面沿用样张确认的同一风格和同一生图后端。

## 阶段 7：质量检查与修复

组装前，agent 会逐页检查文字清晰度、与大纲的一致性、内容截断、风格统一性、多余页码和元素重叠等问题。发现严重问题的页面会用更严格的提示词重新生成，局部小问题优先用图片编辑能力定向修复。

## 阶段 8：演讲稿与组装

生成 `speech.md` 演讲稿，然后用 `assemble_ppt.py` 组装成 `.pptx`，演讲稿会自动写入每页 PPT 的备注区。

## 阶段 9（可选）：保存风格

如果这套 PPT 用的是自定义或调整过的风格，agent 会在最终报告里提示你可以把风格保存到个人风格库，以后直接按名字复用。参见[风格与个人风格库](styles.md)。
